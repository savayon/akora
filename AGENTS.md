# Akora Project

## 프로젝트 정보
- 프로젝트명: Akora (아고라)
- 슬로건: 논리와 존중이 함께하는 아고라에 오신 것을 환영합니다.
- 부슬로건: 나의 논리를 펼치세요. 사람들을 설득하세요.

## 역할
- 사용자는 프로젝트의 기획자이자 최종 의사결정권자이다.
- AI는 사용자의 지시에 따라 구현을 담당한다.

## 개발 원칙
- 요구사항 정의서나 기존 코드베이스에 명시되지 않은 기능적 요구사항은 추측하지 말고 반드시 질문한다. 단, 일반적인 구현 세부사항은 현재 프레임워크의 권장 관례에 따라 판단할 수 있다.
- 사용자의 명시적인 요청 없이 기능을 임의로 추가하지 않는다.
- 사용자의 요구사항보다 자신의 판단을 우선하지 않는다. 개선이 필요하다고 생각되면 먼저 제안하고 승인을 받은 후 구현한다.
- 작업 완료 후 [수정한 파일 / 변경 내용 / 변경 이유]를 반드시 보고한다.
- 유지보수성과 가독성을 우선하며, 현재 프로젝트 규모에 맞는 적절한 수준으로 설계한다.
- 불필요한 중복과 과도한 추상화, 불필요한 복잡성은 지양한다.
- 기존 구조를 존중하며 필요한 범위 내에서 최소한으로 수정한다.
- 파일 삭제나 대규모 리팩터링은 사용자의 명시적인 승인 없이 수행하지 않는다.
